# Справочник: ABI и структура source.zip

Этот документ самодостаточен: on-prem инстанс без доступа к мастеру загружает
скиллы по нему. Не меняется без мажорной версии схемы.

Источники: `services/mcphub/docs/tinygo-subset.md` и
`services/skillfactory/embeddedpack/embed.go` (функция `flatZip`).

---

## ABI гостя (заморожен)

Гостевой WASM-модуль экспортирует ровно три функции:

| Экспорт              | Сигнатура (Wasm)            | Назначение |
|----------------------|-----------------------------|------------|
| `mcphub_abi_version` | `() -> i32`                 | Версия ABI, собранная в модуле; для v1 = `1` |
| `alloc`              | `(size i32) -> i32`         | Выделить гостевой буфер; возвращает `0` при OOM/trap |
| `handle`             | `(ptr i32, len i32) -> i64` | Обработать один запрос; возвращает `(ptr<<32)|len` |

Функция `free(ptr i32, size i32)` **зарезервирована** для будущей фазы —
codegen **не должен** эмитировать функцию с этим именем.

### Host-import: `cap_call`

Хаб регистрирует одну host-функцию, которую импортирует гость:

```
module:   mcphub
function: cap_call
```

Сигнатура (Wasm): `(capPtr u32, capLen u32, argsPtr u32, argsLen u32) -> u64`

Через неё гость вызывает все возможности (http, kv, secret, llm и т.д.).
Возврат: упакованный `(resultPtr<<32)|resultLen`.

## Флаги сборки (заморожены)

```
tinygo build -o out.wasm -target=wasm-unknown -buildmode=c-shared -scheduler=none -panic=trap .
```

> `skillgen check .` запускает ровно эту команду с замороженными флагами — копировать вручную не нужно.

| Флаг                       | Значение |
|----------------------------|---------|
| `-target=wasm-unknown`     | Freestanding Wasm; **WASI выключен**: модуль не должен импортировать `wasi_snapshot_preview1` |
| `-buildmode=c-shared`      | Reactor-модуль: имеет `_initialize` вместо `_start`; хаб вызывает `_initialize` однократно |
| `-scheduler=none`          | Синхронный гость, без планировщика горутин |
| `-panic=trap`              | Паника → wasm `unreachable` trap (WASI не задействован даже в panic-путях) |

Тулчейн: **`tinygo-0.41.1`** (первая ветка 0.41.x, собранная на Go 1.26).
Строка `tinygo-0.41.1` является частью ключа кеша сборки — смена тулчейна инвалидирует
все кешированные артефакты.

## Инварианты гостя

- **Синхронность.** Нет горутин, каналов, `time.Sleep`. Запуск горутины из
  экспортируемой функции паникует (tinygo#3095). `handle` — это чистый
  «запрос → ответ».
- **Нет WASI / нет syscall.** Время, случайность, логирование, HTTP, KV, секреты —
  только через `cap_call`. Прямой доступ к ОС недоступен по конструкции.
- **Per-call инстанс.** Свежий экземпляр модуля создаётся на каждый вызов и
  уничтожается после возврата `handle`; ручной `free` в Phase 1 не нужен.
- **Квоты.** `timeout_s` (wall-clock) и `memory_mb` принудительно применяются
  хабом. `memory_mb` ≤ 4096 (ограничение wazero: `memory.grow` за предел
  возвращает `-1`, последующий out-of-bounds access даёт trap; хаб-процесс выживает).

## Формат конверта (envelope)

Ответ `handle` — JSON-конверт, упакованный через `writeJSON`:

**Успех:**
```json
{"ok": true, "result": <любой JSON>}
```

**Ошибка:**
```json
{"ok": false, "error": {"code": "...", "message": "..."}}
```

Аналогичный конверт используется в ответах `cap_call` (`result`/`error`).
`EnvError.Details` (опциональное поле) — `json.RawMessage`.

## Структура `source.zip`

Хаб принимает **плоский zip**: все файлы лежат в корне архива (без вложенных
директорий). После распаковки хаб запускает `tinygo build .` в этом корне.

### Обязательные файлы

| Имя в zip  | Источник | Описание |
|------------|----------|----------|
| `main.go`  | Автор    | Точка входа; `func main() {}` (пустой) + `handle()` |
| `mcphub_gen.go` | `skillgen gen` | Сгенерированный ABI-boilerplate + типизированные обёртки |
| `go.mod`   | Автор (файл назван `go.mod.txt` в исходнике, переименовывается в zip) | Модульный файл |

### Правила упаковки

- Файлы помещаются **в корень** zip (плоская структура).
- Файл с именем `go.mod.txt` в исходнике переименовывается в `go.mod` при упаковке.
- Файлы `*_test.go` **исключаются**: хабу они не нужны, тесты гоняются локально
  обычным `go test`.
- Поддиректории пропускаются (только файлы верхнего уровня попадают в zip).

### Пример упаковки вручную

```bash
cd source
zip -X ../source.zip main.go mcphub_gen.go go.mod
```

(`go.mod.txt` → `go.mod` переименование нужно только если ваш исходник хранит
файл под именем `go.mod.txt`; при ручной упаковке добавляйте уже под именем `go.mod`.)

### Логика `flatZip` (справочно)

`flatZip` в `embeddedpack` делает следующее:
1. Читает записи директории `srcDir` (не рекурсивно).
2. Пропускает поддиректории (`e.IsDir() → skip`).
3. Пропускает файлы с суффиксом `_test.go`.
4. Переименовывает `go.mod.txt` → `go.mod`.
5. Добавляет оставшиеся файлы в zip плоско (без path-prefix).

## go.mod гостевого модуля

```
module mcphubguest/<bundle-name>

go 1.26
```

Внешних зависимостей быть не должно: весь runtime предоставляется через
`cap_call`. `encoding/json` и `unsafe` входят в stdlib TinyGo.
